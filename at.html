Questão 1

Quando é recomendável a criação de um site separado para mobile? Seria este o caso do site integrado de Interfaces - SPA?
Atualmente é recomendado que todo e qualquer site tenha uma versão mobile, uma vez que uma grande parte dos acessos são provenientes de aparelhos smartphones. Para evitar a necessidade de criação de duas páginas web distintas, é recomendado a criação de sites responsivos, que conseguem adequar seu conteúdo dependendo do tamanho da tela do usuário. Esse tipo de abordagem é recomendada para todo e qualquer site, não somente Single Page Aplication (SPA). No entanto, se tratando de um site com volume de informações e com grande retorno financeiro, já é recomendado a criação de um App dedicado, pois usa usabilidade e performance tendem a ser melhores que site responsivo. Além disso, para alguns projetos, que demandam alguns requisitos, somente um app terá condições de atender.

Ex.:

Questão 3

O elemento áudio permite como elemento‐filho o elemento source. Quais são os atributos que o source permite e qual conteúdo eles suportam? Quais os outros atributos do elemento áudio e quando devem ser utilizados?
O atributo source do elemento áudio, permite que você indique o caminho relativo do arquivo de áudio que deseja. Atualmente, existem três tipos de formatos de áudios suportados em HTML5: MP3, WAV, and OGG. Além do atributo source, o elemento áudio também possui outros atributos como:
autoplay: Especifica se o áudio deve ser tocado automaticamente assim que os controles estiverem prontos.
controls: Utilizado para especificar se os controles, como pause e play, devem ficar disponíveis.
loop: Utilizado para especificar se o áudio deve começar a tocar novamente uma vez que ele termina.
muted: Utilizado para especificar se o áudio deve se inicializar sem som.
preload: Utilizado para especificar como o áudio deve ser pré-carregado no carregamento da página.

Questão 4

O elemento vídeo permite como elemento‐filho o elemento source. Quais são os atributos que o source permite e qual conteúdo eles suportam? Quais os outros atributos do elemento video e quando devem ser utilizados?
O atributo source do elemento vídeo, permite que você indique o caminho relativo do arquivo de vídeo que deseja. Atualmente, existem três tipos de formatos de vídeo suportados em HTML5: MP4, WebM, and OGG. Além do atributo source, o elemento vídeo também possui outros atributos como:
autoplay: Especifica se o vídeo deve ser tocado automaticamente assim que os controles estiverem prontos.
controls: Utilizado para especificar se os controles, como pause e play, devem ficar disponíveis.
loop: Utilizado para especificar se o vídeo deve começar a tocar novamente uma vez que ele termina.
muted: Utilizado para especificar se o vídeo deve se inicializar sem som.
preload: Utilizado para especificar como o vídeo deve ser pré-carregado no carregamento da página.
poster: Utilizado para especificar uma imagem que será exibida enquanto o vídeo está sendo baixado, ou antes do usuário pressionar o botão de play.
height: Utilizado para especificar a altura do vídeo.
width: Utilizado para especificar a largura do vídeo.

Questão 5

No caso do canvas, HTML 5 nunca, em momento algum, fez qualquer animação. Isso porque o HTML5 é uma simples linguagem de marcação. Dessa forma, o que é necessário para criar animações com HTML5? O que são APIs do HTML5?
O elemento canvas funciona como um quadro no qual dentro podem ser exibidos vários frames de animação. Para construir essas animações é necessário utilizar CSS e JavaScript em adição ao HTML. Para construir essas animações também pode-se utilizar de outras ferramentas, APIs, que no qual já possuem funções pré definidas que simplificam o processo de criação de animações. Utilizando APIs é possível criar códigos menores utilizando funções que nelas são fornecidas e que ao final são convertidas, de maneira automática pela API, em um código HTML, CSS e JS simples para a leitura do browser.
